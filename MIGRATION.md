# 🔄 Гайд миграции со старой версии

Если у тебя уже был бот с файлом `main.py` на 3000 строк, вот как перейти на новую модульную версию **без потери данных**.

## ⚠️ Перед миграцией

1. **Сделай бэкап** текущей БД:
   ```bash
   cp hupha_bot.db hupha_bot.db.backup
   ```

2. **Останови текущий бот**

## 📋 Процесс миграции

### Шаг 1: Замени структуру

Замени старый монолитный бот на новую модульную структуру:
- Удали старый `main.py` (или переименуй в `main.py.old`)
- Скопируй папку `bot/` из новой версии
- Скопируй новые `main.py` (точка входа) и `README.md`

**После этого структура должна быть:**
```
bot/
  config.py
  state.py
  utils.py
  db/
    database.py
    __init__.py
  ai/
    ai_client.py
    rp.py
    __init__.py
  handlers/
    commands.py
    economy.py
    trade.py
    messages.py
    __init__.py
  main.py

main.py          ← точка входа
requirements.txt
.env
lore.txt
hupha_bot.db     ← твоя старая БД (совместима!)
```

### Шаг 2: Проверь `.env`

Убедись, что `.env` содержит:
```
BOT_TOKEN=...
GROQ_KEY=...
ADMIN_ID=...
```

### Шаг 3: Запусти новый бот

```bash
python main.py
# или
python -m bot.main
```

**Что произойдёт:**
1. ✓ Старая `hupha_bot.db` прочитается полностью — все игроки, вики, рп-сессии, достижения сохранятся
2. ✓ Будет создана новая `bot_state.db` для состояний диалогов
3. ✓ Бот запустится нормально

### Шаг 4: Тест

1. Напиши в бот `/start` — должна появиться обновлённая надпись
2. Напиши `/getid` — должен вывести твой ID
3. Проверь, что `/pay` и `/craft` доступны (новые команды)
4. Попробуй купить что-то в магазине

## 🔄 Обратная совместимость

**Данные которые сохранятся:**
- ✓ Все игроки и их профили
- ✓ База знаний (wiki)
- ✓ Магазин (items)
- ✓ Истории
- ✓ РП-сессии
- ✓ Достижения
- ✓ Все таблицы, которые уже были

**Состояния, которые потеряются (это нормально):**
- ✗ Прерванные сценарии регистрации (user_states) — игроки просто переделают
- ✗ Активные РП-сессии в памяти — но рабочая история сохранится в БД
- ✗ Временные данные при обучении вики

**Это неважно**, потому что:
1. Большинство сценариев короткие (1-2 шага)
2. Важные данные (профили, предметы, ачивки) уже в БД
3. Следующая такая же миграция их уже восстановит (thanks to `bot_state.db`)

## 🆘 Если что-то сломалось

### "ModuleNotFoundError: No module named 'bot'"

Убедись, что запускаешь из корня проекта:
```bash
cd /path/to/bot
python main.py
# Не делай: cd /path/to/bot/bot && python main.py
```

### "No such table: ..."

Миграция не сработала. Попробуй:
```bash
rm bot_state.db  # Удалить старое (если было)
python main.py   # Пересоздаст пустую
```

Если проблема в hupha_bot.db — восстанови из бэкапа:
```bash
rm hupha_bot.db
cp hupha_bot.db.backup hupha_bot.db
python main.py
```

### Состояния не восстанавливаются после рестарта

Это ожидаемо для первого рестарта (сейчас впервые сохраняем).
Попробуй ещё раз:
1. Запусти бота
2. Начни какой-то сценарий (например, `/create` — вводи имя)
3. Перезагрузи бота (`Ctrl+C`, потом `python main.py`)
4. При следующем `/create` должно восстановиться

## 📊 Дополнительно

### Если хочешь использовать старые данные в других местах

Файлы `hupha_bot.db` полностью совместимы с SQLite:
```bash
sqlite3 hupha_bot.db
sqlite> .tables
sqlite> SELECT * FROM players LIMIT 3;
sqlite> SELECT COUNT(*) FROM wiki;
```

### Если нужно мигрировать на другой бот/язык

Экспортируй из БД:
```bash
sqlite3 hupha_bot.db ".mode csv" ".output backup.csv" "SELECT * FROM players;"
# Потом импортируй в другую систему
```
