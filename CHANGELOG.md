# 📝 Changelog v2.0

## ✨ Новые фичи

### 💸 Перевод валюты между игроками (`/pay`)

**Команда:** `/pay сумма [рубли|хуфа]` (в ответ на сообщение игрока)

Или явно: `/pay id сумма [рубли|хуфа]`

**Примеры:**
```
/pay 50 рубли        # Переводит 50 рублей текущему собеседнику
/pay 123456 100 хуфа # ID 123456 получает 100 хуфы
```

**Особенности:**
- Проверка баланса (не даст вывести больше, чем есть)
- Получатель уведомляется в ЛС
- Срабатывают достижения за активность
- Работает с обеими валютами: рубли и хуфа

### ⚒️ Крафт предметов (`/craft`)

**Команда:** `/craft предмет1, предмет2`

Объединяет два предмета из инвентаря в новый. Цена нового ≈ 70% суммы исходных.

**Примеры:**
```
/craft меч, щит
/craft золото, кожа
```

**Особенности:**
- Просто работает с существующими items/inventory — не требует таблицы рецептов
- Можно комбинировать одинаковые предметы (если 2+ штуки)
- Новый предмет получает имя и описание
- Добавляется в инвентарь автоматически

## 💾 Постоянное хранение состояний (самое важное)

### Проблема старой версии
- user_states, temp_data, dialogue_learning и т.д. жили только в памяти
- При рестарте бота → потеря всех прерванных диалогов
- Если игрок дописывает анкету в 3 шага, и на шаге 2 бот упал → начинать с начала

### Решение
**Новая таблица `bot_state.db`:**
- Автосохранение каждые 15 секунд в фоновом потоке
- Восстановление при старте бота
- Гарантирует, что игрок продолжит там, где закончил

**Реализация:** `bot/state.py`
```python
load_state()      # При старте — восстанавливаем
start_autosave()  # Запускаем фоновый поток сохранения
save_state()      # Можно вызвать явно, вызывается при SIGTERM/SIGINT
```

**Как это работает:**
1. Все state-dict'ы (user_states, temp_learning и т.д.) регистрируются в `_PERSISTED`
2. Каждые 15 сек фоновый поток вызывает `save_state()` → сериализует в JSON → пишет в БД
3. При старте `load_state()` читает из БД → восстанавливает dict'ы

## 📦 Потребление API

### Groq API
- **Модель:** `llama-3.1-8b-instant` (free tier)
- **Всё ещё быстро** потому что мы не поменяли модель
- **Потребление:**
  - `/npc`, `/quest`, `/location` — 1 запрос
  - `/oracle` с интерпретацией — 1 запрос
  - `polish_rp_text` — 1 запрос (можно отключить, если лимит)
  - Каждый запрос ~500-1500 tokens

**Совет:** Если лимит быстро кончается:
1. Убрать `polish_rp_text()` (просто возвращать text как есть)
2. Убрать автокатегоризацию вики (`ai_categorize_keyword`)
3. Кэшировать результаты генераторов в БД

## 🐛 Известные ограничения

1. **RateLimiting:** Groq free tier имеет лимиты — если много игроков одновременно, может быть throttling
2. **SQLite:** Для большого (100k+) количества игроков лучше миграция на PostgreSQL
3. **Persistent state:** Снапшоты в JSON не сжимаются — очень активные диалоги могут занять место в `bot_state.db`
