# 🕯 Библиотека Хуфы — переработанная версия

Бот для Telegram с РП-сессиями, AI-генератором контента и экономикой.

## 🔄 Что изменилось


### ✅ Постоянное хранение состояний (не теряются при рестарте)

Раньше: `user_states`, `temp_data`, `dialogue_learning` и т.д. жили в памяти процесса.
При рестарте бота (деплой, обновление, падение) — всё слетало.

**Теперь:**
- Все состояния автоматически сохраняются в `bot_state.db` каждые 15 секунд.
- При старте бота — восстанавливаются из БД.
- Игрок может не дописать анкету → перезагрузить бота → продолжить с того же места.

**Реализация:** `bot/state.py`
- `load_state()` — восстанавливает при старте
- `start_autosave()` — фоновый поток + сигналы SIGTERM/SIGINT
- `save_state()` — снимает снапшот в `bot_state.db`

### ✅ Экономика: перевод валюты между игроками (`/pay`)

```
/pay сумма [рубли|хуфа]      # Ответ на сообщение игрока
/pay id сумма [рубли|хуфа]   # Или явно укажи ID
```

Примеры:
```
/pay 50 рубли        # (ответ на его сообщение) Переводит 50 рублей
/pay 123456 100 хуфа # ID 123456 получит 100 хуфы
```

**Особенности:**
- Проверяет баланс — не допустит вывести больше, чем есть
- Уведомляет получателя в личные сообщения
- Срабатывают достижения за активность

### ✅ Крафт: объединять предметы в новые (`/craft`)

```
/craft меч, щит
```

Система без отдельной таблицы рецептов:
- Берёт два предмета из инвентаря
- Удаляет исходные (по 1 штуке каждого)
- Создаёт новый предмет ✨ (цена ≈ 70% суммы исходных)
- Добавляет в инвентарь

Примеры: `меч + щит = меч + щит` (новая комбо), `золото + кожа = золотая броня` и т.д.

## 🚀 Установка и запуск

### 1. Клонировать / распаковать проект

```bash
cd /path/to/bot
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Заполнить `.env`


```bash

```

```bash
python main.py
```

## 📦 Как расширять

### Добавить новую команду

1. **Создать обработчик** в `bot/handlers/` (или добавить в существующий файл):
   ```python
   from bot.config import bot, ADMIN_ID
   from bot.db.database import execute_db
   
   @bot.message_handler(commands=['mycommand'])
   def my_command(message):
       uid = message.from_user.id
       # ... логика ...
   ```

2. **Обработчик автоматически зарегистрируется** благодаря импорту в `bot/handlers/__init__.py`

### Добавить новую ИИ-функцию (генератор NPC, квеста и т.д.)

1. **Добавить в `bot/ai/ai_client.py`** (или создать новый файл, если станет слишком велико):
   ```python
   def generate_my_stuff(param1, param2, context=""):
       system = "Ты — генератор крутых штук. ..."
       user = f"Создай крутую штуку с {param1}..."
       return groq_complete(system, user, temperature=0.7, max_tokens=1500)
   ```

2. **Импортировать в обработчик**:
   ```python
   from bot.ai.ai_client import generate_my_stuff
   ```

### Добавить новую таблицу в БД

1. **Добавить CREATE TABLE в `init_db()`** (`bot/db/database.py`):
   ```python
   execute_db('''CREATE TABLE IF NOT EXISTS my_table (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       name TEXT NOT NULL
   )''')
   ```

2. **Добавить хелпер-функции** для работы с таблицей в том же файле

3. **Импортировать в обработчики**:
   ```python
   from bot.db.database import my_table_function
   ```

### Сохранить новое состояние диалога при рестарте

1. **Добавить dict** в `bot/state.py`:
   ```python
   my_state = {}
   ```

2. **Добавить в `_PERSISTED`**:
   ```python
   _PERSISTED = {
       'my_state': my_state,
       # ...
   }
   ```

3. **Использовать как обычный dict** — автосохранение работает само

## 🔧 Потребление Groq API

Бот использует **free tier** модели `llama-3.1-8b-instant`:
- Очень быстро (идеально для РП-сессий)
- Достаточно смарт для генерации (NPC, локации, квесты)
- Имеет fair-use лимиты

**Совет:** Если хочешь снизить лимит ещё больше:
1. **Убрать `polish_rp_text()`** в `bot/ai/rp.py` — просто возвращать текст как есть
2. **Убрать автокатегоризацию** — задавать категории вручную
3. **Кэшировать результаты** генераторов в БД

## 📝 Структура данных

### Таблицы игровые (hupha_bot.db)

| Таблица | Назначение |
|---------|-----------|
| `players` | Анкеты игроков (имя, био, баланс валют, достижения) |
| `wiki` | База знаний (статьи) |
| `items` | Каталог товаров/предметов |
| `inventory` | Инвентарь игрока (кто, какой предмет, количество) |
| `stories` | Истории, поделённые на части |
| `rp_sessions` | РП-сессии (кто ГМ, контекст, статус) |
| `achievements` | Определения достижений |
| `player_achievements` | Какие ачивки разблокировал игрок |

### Таблица состояний (bot_state.db)

| Таблица | Назначение |
|---------|-----------|
| `bot_state` | Снапшоты всех state-dict'ов (user_states, temp_data, rp_sessions и т.д.) |
